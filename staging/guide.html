3처음 한 번 — 연결·웹훅 설정 ↑ 위로

3-1. 시트 연결

아무 페이지에서든 구글시트 URL을 한 번 연결하면 모든 페이지가 같은 시트를 공유합니다(브라우저에 저장). 시트는 "링크가 있는 모든 사용자 — 뷰어"로 공유되어 있어야 읽을 수 있습니다.

3-2. 쓰기 웹훅(Apps Script) — 자동 반영을 켜려면

사이트는 정적 페이지라 시트에 직접 쓸 수 없습니다. 아래 Apps Script를 1회 배포해 "쓰기 통로"를 열면, 이후엔 엑셀 드롭 → 반영이 자동으로 됩니다. (설정 전에도 CSV 내려받기로 수동 반영 가능)

  1. 구글시트 → 상단 메뉴 확장 프로그램 → Apps Script
  2. 기본 코드를 지우고 아래 코드를 붙여넣고 저장(💾)
  3. 우측 상단 배포 → 새 배포 → 유형 웹 앱 선택
  4. 실행 계정 나, 액세스 권한 모든 사용자 → 배포 (최초 1회 권한 승인)
  5. 생성된 웹 앱 URL(…/exec)을 복사
  6. 업로드 페이지 → "쓰기 웹훅"에 붙여넣고 저장
function doPost(e){
  try{
    var b = JSON.parse(e.postData.contents);
    var ss = SpreadsheetApp.getActiveSpreadsheet();
    var sh = ss.getSheetByName(b.sheet);
    if(!sh){ sh = ss.insertSheet(b.sheet); sh.appendRow(b.header); }
    var ncol = b.header.length;
    var repl = {}; (b.replaceDates||[]).forEach(function(d){ repl[d] = true; });
    var last = sh.getLastRow(), keep = [];
    if(last > 1){
      var data = sh.getRange(2,1,last-1,ncol).getValues();
      for(var i=0;i<data.length;i++){ if(!repl[iso_(data[i][0])]) keep.push(data[i]); }
    }
    var incoming = (b.rows||[]).map(function(r){ var row=r.slice(); row[0]=parseDate_(r[0]); return row; });
    var merged = keep.concat(incoming);
    merged.sort(function(a,b){ return a[0]-b[0]; });
    if(last > 1) sh.getRange(2,1,last-1,ncol).clearContent();
    if(merged.length) sh.getRange(2,1,merged.length,ncol).setValues(merged);
    return json_({ok:true, added:incoming.length, kept:keep.length});
  }catch(err){ return json_({ok:false, error:String(err)}); }
}
function parseDate_(iso){ var p=String(iso).split("-"); return new Date(+p[0], +p[1]-1, +p[2]); }
function iso_(v){
  if(v instanceof Date) return Utilities.formatDate(v, Session.getScriptTimeZone(), "yyyy-MM-dd");
  var d = new Date(String(v)); return isNaN(d) ? String(v) : Utilities.formatDate(d, Session.getScriptTimeZone(), "yyyy-MM-dd");
}
function json_(o){ return ContentService.createTextOutput(JSON.stringify(o)).setMimeType(ContentService.MimeType.JSON); }
이 코드는 대상 날짜 행만 제거 후 파일 행을 추가하고 날짜순으로 재정렬합니다. 파일에 없는 과거 날짜는 절대 건드리지 않습니다.

4주기 업무 — 데이터 업데이트 ↑ 위로

  1. 사내 지표에서 각 프로젝트 엑셀 다운로드 — 결제사·플랫폼별 · daily_kpi 자체결제 · daily_kpi(순수) 파일 (세 유형 모두 자동 인식)
  2. 업로드 페이지에 여러 개 한 번에 드롭 (파일명 앞부분으로 프로젝트 자동 인식)
  3. 파일별로 신규 N일 변경 M일 동일 K일 자동 표시 → "변경 상세"로 기존→신규 매출 확인
  4. 변경분 반영 버튼 클릭 → 신규+변경 날짜만 시트에 반영
  5. 대시보드/플래너/전사비교를 새로고침하면 반영됨
30일 규칙 — 다운로드가 최근 30일치만 담기므로, 업로드 간격이 30일을 넘기면 그 사이 날짜가 비어버립니다. 최소 30일에 한 번은 올려주세요.
덮어쓰기 걱정 없음 — 겹치는 날은 값이 바뀐 경우에만 교체(정산 지연 수정치 반영), 동일하면 건너뜁니다. 과거 이력은 보존됩니다.

6지표 정의 ↑ 위로

지표정의
자체결제결제수단(PG)이 다날·엑솔라·페이레터·페이레터 PPAY인 매출 (마켓 라벨 Google/Apple과 무관)
일평균 PU월 PU합 ÷ 일수 (PU는 채널 합산 기준)
ARPPUIAP 매출 ÷ PU (광고 IAA 제외)
자체결제 매출비중자체결제 매출 ÷ 전체 IAP 매출
자체결제 PU비중자체결제 일평균 PU ÷ 전체 일평균 PU
PU 기준 두 가지 — 결제사 탭 기반 지표(대시보드·플래너·자체결제 PU)는 채널 합산(여러 PG 결제 중복 포함). 전사 비교의 전체 PU·ARPPU는 daily_kpi 순결제자 기준(해당 월 코드-daily 데이터가 있을 때, 없으면 채널합산 폴백). 그래서 전사비교 전체 PU가 결제사 합산보다 작을 수 있습니다.

8문제 해결 (FAQ) ↑ 위로

데이터가 안 보여요 / "데이터 미연결"
시트 URL 연결 여부와 시트 공유 설정("링크가 있는 모든 사용자 — 뷰어")을 확인하세요. 대시보드·플래너에서 한 번 연결하면 다른 페이지도 이어집니다.
업로드했는데 "동일 N일"만 나와요
정상입니다. 이미 시트에 있는 날짜와 값이 같으면 반영할 게 없습니다. 값이 바뀐 날만 변경으로 잡힙니다.
"프로젝트 미인식"
파일명 앞부분이 프로젝트 코드가 아니면 못 잡습니다. 파일 카드의 드롭다운에서 직접 선택하세요.
웹훅 반영이 실패해요
Apps Script를 "모든 사용자" 액세스로 배포했는지, 웹 앱 URL이 /exec로 끝나는지 확인하세요. 코드 수정 후에는 새 배포(또는 배포 관리에서 버전 갱신)가 필요합니다. 임시로는 CSV 내려받기로 반영하세요.
중간 날짜가 비어요
다운로드가 최근 30일치라, 업로드 간격이 30일을 넘기면 공백이 생깁니다. 30일 안에 한 번씩 올리세요.
